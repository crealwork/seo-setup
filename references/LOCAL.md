# LOCAL — 로컬 SEO (오프라인 매장 · 지역 서비스만)

검색엔진 등록만큼 '지도·지역 정보 서비스' 등록이 중요하다. 핵심 원리 하나:
**리뷰는 검색엔진의 데이터 소스다** — 내가 쓴 소개글보다 고객 리뷰 속 단어가
노출을 결정한다 (구글·네이버 공통).

## 1. Google Business Profile (Google Maps)

- **리뷰 = 랭킹 데이터**: 소개글에 '영어 진료'가 없어도 고객 리뷰에 "영어 진료가
  가능해서 편해요"가 있으면 [강남 영어진료 병원] 검색에 노출된다.
- **구체적 리뷰 유도**: "맛있어요/최고예요"는 데이터 가치가 없다. "쌀빵 진짜
  맛있어요, 아기랑 오기 좋아요"처럼 **메뉴명 + 상황**이 담기게 유도 (후기 사은품
  이벤트를 쓴다면 리뷰 가이드 문구에 메뉴명·상황 예시를 넣어라).
- **언어별 노출**: 구글은 언어별로 따로 분석 — 외국인 고객의 영어 리뷰('Vegan
  Bagel')가 영어 검색 유저 노출을 만든다.
- **사장님 답글로 키워드 보강**: 리뷰에 핵심 키워드가 빠졌으면 답글에서
  "저희 '비건 베이글'을 좋아해 주셔서…"로 단서를 채운다.

## 2. Naver Place (한국)

랭킹 = '네이버가 신뢰할 데이터'를 얼마나 촘촘히 주느냐.

**운영 세팅**
- 정보 풀 세팅 + **스마트콜·네이버 예약·톡톡·쿠폰 전부 활성화** — 활동성 높은
  매장이 상단에 뜬다.
- 재방문 선순환: 리뷰 답글, 재방문 쿠폰, 톡톡 마케팅.
- '맛집' 같은 메인 키워드보다 **순위 유지가 쉬운 뾰족한 세부 키워드 여러 개**로
  매출 베이스를 깐다.

**대표 키워드 5개 (각 15자) 세팅 룰**
- 네이버는 키워드를 **교차 조합**해 인식 — 1번과 5번의 단어가 합쳐져 검색에
  반영된다. 따라서 **중복 단어 금지**, 최대한 다양한 단어 배치.
- 공식 행정명은 자동 인식 — 넣지 마라. 대신 **줄임말·구 명칭·"OO대 앞"** 같은
  비공식 지명은 수동으로 넣어야 잡힌다.
- 등록 후 **직접 검색해서 형태소 분해가 의도대로인지 확인**.
- 공급자 마인드 버리기: 메뉴명이 '버럴떡'이어도 고객이 '버터떡'으로 검색하면
  키워드는 '버터떡'.

## 3. Kakao Maps (한국)

- 카카오톡 '장소 공유'·검색과 연동 — 카톡 내비·택시 등 카카오 생태계 전체에
  위치 정보 노출.

## 4. Yelp (북미)

- **Apple Maps와 Yahoo 로컬 데이터의 원천** — 등록 안 하면 아이폰 유저가
  Siri/지도로 찾을 때 정보가 누락된다. 북미 로컬 서비스는 방어적으로라도 등록.
